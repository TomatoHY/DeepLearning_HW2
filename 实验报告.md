# Task 1: 图像分类

## 一、实验概述

本任务使用 CIFAR-10 数据集进行图像分类，实现并对比了三种不同的神经网络架构：
- **MLP (多层感知机)**: 全连接网络
- **CNN (卷积神经网络)**: 标准卷积架构
- **ResNet (残差网络)**: 带残差连接的深度网络

## 二、数据集

**CIFAR-10 数据集**:
- 训练集: 50,000 张图像
- 测试集: 10,000 张图像
- 图像尺寸: 32×32 RGB
- 类别数: 10 (飞机、汽车、鸟、猫、鹿、狗、青蛙、马、船、卡车)

**数据增强**:
- 随机水平翻转
- 随机裁剪 (padding=4)
- 归一化: mean=[0.4914, 0.4822, 0.4465], std=[0.2470, 0.2435, 0.2616]

## 三、模型架构

### 3.1 MLP
```
Input (3072) → FC(512) → ReLU → Dropout(0.5) 
→ FC(256) → ReLU → Dropout(0.5) 
→ FC(10)
```
- 参数量: ~1.8M
- 特点: 简单但忽略空间结构

### 3.2 CNN
```
Conv(64) → Conv(64) → MaxPool 
→ Conv(128) → Conv(128) → MaxPool 
→ Conv(256) → Conv(256) → MaxPool 
→ FC(512) → FC(10)
```
- 参数量: ~2.3M
- 特点: 利用卷积提取空间特征

### 3.3 ResNet
```
Conv(64) → ResBlock×3 → ResBlock×3 → ResBlock×3 
→ AvgPool → FC(10)
```
- 参数量: ~0.5M
- 特点: 残差连接缓解梯度消失

## 四、训练配置

| 参数 | 值 |
|------|-----|
| Optimizer | SGD with momentum (0.9) |
| Learning Rate | 0.1 |
| LR Schedule | StepLR (γ=0.1, step=30) |
| Batch Size | 128 |
| Epochs | 100 |
| Weight Decay | 5e-4 |
| Loss Function | CrossEntropyLoss |

# Task 2: 目标检测

## 一、实验概述

本任务在 PASCAL VOC 2007 数据集上实现目标检测，使用 Faster R-CNN 架构，基于 ResNet-50 预训练模型进行微调。

## 二、数据集

**PASCAL VOC 2007**:
- 训练集: 2,501 张图像
- 验证集: 2,510 张图像
- 测试集: 4,952 张图像
- 类别数: 20 (人、动物、交通工具、家具等)

**数据增强**:
- 随机水平翻转 (p=0.5)
- 归一化: ImageNet 统计量

## 三、模型架构

**Faster R-CNN with ResNet-50 FPN**:

```
Backbone: ResNet-50 (预训练)
    ↓
Feature Pyramid Network (FPN)
    ↓
Region Proposal Network (RPN)
    ↓
RoI Align + Detection Head
    ↓
Classification + Bounding Box Regression
```

**关键组件**:
- **Backbone**: ResNet-50 提取多尺度特征
- **FPN**: 融合不同层级特征，处理多尺度目标
- **RPN**: 生成候选区域
- **RoI Align**: 精确特征对齐
- **Detection Head**: 分类 + 边界框回归

- Epoch 1-3: lr = 0.005
- Epoch 4-6: lr = 0.0005
- Epoch 7-9: lr = 0.00005
- Epoch 10-12: lr = 0.000005

## 六、结果分析

### 6.1 性能特点

**优势**:
- ✓ 高精确率 (94.95%): 检测到的目标大多正确
- ✓ 训练稳定: 损失平稳下降
- ✓ 预训练有效: ResNet-50 提供良好初始化

**不足**:
- ✗ 低召回率: 检测准确率仅 19.52%
- ✗ 漏检严重: 大量目标未被检测到

### 6.2 问题分析

**低召回率原因**:
1. **训练不足**: 仅 12 个 epoch，模型未充分学习
2. **数据量小**: VOC 2007 训练集较小 (2,501 张)
3. **学习率衰减过快**: 后期学习率过小，优化停滞
4. **RPN 阈值**: 可能过于保守，生成候选框不足

**高精确率原因**:
1. **预训练权重**: ResNet-50 提供强大特征提取能力
2. **保守检测**: 模型倾向于只输出高置信度预测

### 6.3 改进建议

1. **延长训练**: 增加至 20-30 个 epoch
2. **调整学习率**: 使用更温和的衰减策略
3. **数据增强**: 添加更多增强方法 (色彩抖动、缩放等)
4. **调整 RPN**: 降低 NMS 阈值，增加候选框数量
5. **损失权重**: 调整分类和回归损失的平衡

# Task 3: 语义分割

## 一、实验概述

本任务在 Stanford Background 数据集上实现语义分割，使用 U-Net 架构，对比了三种损失函数的效果。

## 二、数据集

**Stanford Background Dataset**:
- 训练集: 600 张图像
- 验证集: 115 张图像
- 图像尺寸: 320×240 → 256×256
- 类别数: 8 (天空、树、路、草、水、建筑、山、前景物体)
- 特殊标签: -1 (未标注区域，训练时忽略)

**数据预处理**:
- Resize 到 256×256
- 归一化: mean=[0.485, 0.456, 0.406], std=[0.229, 0.224, 0.225]
- 标签处理: 保持原始整数值，ignore_index=-1

## 三、模型架构

**U-Net (31.04M 参数)**:

```
Encoder (下采样):
  Conv(64) → Conv(64) → MaxPool
  Conv(128) → Conv(128) → MaxPool
  Conv(256) → Conv(256) → MaxPool
  Conv(512) → Conv(512) → MaxPool
  
Bottleneck:
  Conv(1024) → Conv(1024)
  
Decoder (上采样):
  UpConv(512) + Skip → Conv(512) → Conv(512)
  UpConv(256) + Skip → Conv(256) → Conv(256)
  UpConv(128) + Skip → Conv(128) → Conv(128)
  UpConv(64) + Skip → Conv(64) → Conv(64)
  
Output:
  Conv(8) → Softmax
```

**关键特性**:
- **跳跃连接**: 融合编码器和解码器特征
- **对称结构**: 编码器和解码器镜像对称
- **多尺度特征**: 捕获不同层级的语义信息

## 四、训练配置

### 4.1 基础配置

| 参数 | 值 |
|------|-----|
| Optimizer | Adam |
| Learning Rate | 1e-4 |
| LR Schedule | ReduceLROnPlateau (factor=0.5, patience=5) |
| Batch Size | 8 |
| Epochs | 100 |
| 评估指标 | mIoU (mean Intersection over Union) |

### 4.2 损失函数对比

实验对比了三种损失函数:

1. **Cross-Entropy Loss**:
   ```python
   loss = CrossEntropyLoss(ignore_index=-1)
   ```

2. **Dice Loss** (手动实现):
   ```python
   dice = 2 * (pred ∩ target) / (pred + target)
   loss = 1 - dice
   ```
   - 直接优化 IoU 相关指标
   - 处理类别不平衡

3. **Combined Loss**:
   ```python
   loss = 0.5 * CE_loss + 0.5 * Dice_loss
   ```
   - 结合两者优势

## 六、结果分析

### 6.1 损失函数对比

**Cross-Entropy Loss**:
- ✓ 训练 mIoU 高 (76.90%)
- ✓ 优化稳定，收敛快

- ✓ **最佳测试性能** (56.94% mIoU)

- ✓ 训练更稳定
- ✗ 训练 mIoU 较低 (68.10%)

**Combined Loss**:
- ✓ 训练 mIoU 最高 (77.93%)
- ✗ 过拟合最严重 (22.42% 差距)
- ✗ 测试性能中等 (56.46%)

2. **类别不平衡处理**: Dice Loss 在困难类别 (water, mountain) 上显著优于其他方法
3. **过拟合问题**: CE 和 Combined Loss 都存在明显过拟合
4. **训练稳定性**: Dice Loss 训练曲线更平滑

# 总结与对比

## 一、三个任务对比

| 任务 | 数据集 | 模型 | 最佳性能 | 主要挑战 |
|------|--------|------|---------|---------|
| 分类 | CIFAR-10 | ResNet | 82.13% | 过拟合、细粒度区分 |
| 检测 | VOC 2007 | Faster R-CNN | 19.52% (Acc) / 94.95% (Prec) | 低召回率、训练不足 |
| 分割 | Stanford BG | U-Net + Dice Loss | 56.94% (mIoU) | 过拟合、类别不平衡 |

## 二、共同经验

### 2.1 成功因素

1. **预训练权重**: 显著提升性能和训练速度 (Task 2)
2. **数据增强**: 有效减少过拟合 (Task 1)
3. **学习率调度**: 帮助模型更好收敛 (所有任务)
4. **架构选择**: 合适的架构对性能至关重要
5. **损失函数**: 针对任务特点选择合适的损失函数 (Task 3)

### 2.2 共同挑战

1. **过拟合**: 所有任务都存在训练/验证性能差距
2. **数据量**: 相对较小的数据集限制了性能
3. **超参数调优**: 需要大量实验找到最佳配置
4. **计算资源**: 训练时间和资源限制
5. **类别不平衡**: 影响模型对少数类的学习

## 三、技术要点

### 3.1 分类任务
- 残差连接对深度网络至关重要
- 参数效率: ResNet 用更少参数达到更好效果
- 混淆矩阵揭示了类别间的相似性
- 数据增强显著提升泛化能力

### 3.2 检测任务
- 预训练权重提供强大的特征提取能力
- 高精确率但低召回率表明模型过于保守
- 需要更长的训练时间和更多数据
- FPN 对多尺度目标检测至关重要

### 3.3 分割任务
- Dice Loss 在处理类别不平衡时表现最好
- U-Net 的跳跃连接对恢复空间细节至关重要
- 损失函数的选择显著影响泛化能力
- 直接优化评估指标 (IoU) 的损失函数效果更好

## 四、未来改进方向

### 4.1 模型层面
- 尝试更先进的架构 (Transformer, EfficientNet)
- 使用更大的预训练模型
- 探索自监督学习方法
- 应用注意力机制

### 4.2 训练层面
- 实现更强的数据增强 (MixUp, CutMix, AutoAugment)
- 使用更复杂的学习率调度 (Cosine Annealing, Warmup)
- 应用知识蒸馏技术
- 使用更长的训练周期

### 4.3 工程层面
- 混合精度训练加速
- 分布式训练处理更大数据集
- 自动化超参数搜索 (Optuna, Ray Tune)
- 模型压缩和量化

### 4.4 任务特定改进

**分类**:
- 使用 Label Smoothing
- 尝试 CutOut、MixUp 等增强
- 增加模型深度和宽度

**检测**:
- 延长训练至 20-30 epochs
- 调整 RPN 参数提高召回率
- 使用 Focal Loss 处理类别不平衡
- 尝试更先进的检测器 (YOLO, DETR)

**分割**:
- 使用 CRF 后处理优化边界
- 尝试 Attention U-Net
- 应用测试时增强 (TTA)
- 调整 Combined Loss 的权重比例

## 五、总结

本次作业完成了三个经典的计算机视觉任务，从简单的图像分类到复杂的目标检测和语义分割。通过实验，我们深入理解了：

1. **架构的重要性**: 不同任务需要不同的网络设计
   - 分类: 残差连接提升深度网络性能
   - 检测: FPN 处理多尺度目标
   - 分割: 跳跃连接恢复空间细节

2. **损失函数的影响**: 损失函数的选择直接影响模型性能
   - Dice Loss 在类别不平衡的分割任务中表现最佳
   - 直接优化评估指标的损失函数效果更好

3. **过拟合的普遍性**: 需要多种正则化技术
   - 数据增强、Dropout、Weight Decay
   - 早停和学习率调度

4. **预训练的价值**: 迁移学习显著提升性能
   - 特别是在数据量较小的情况下

5. **实验设计的重要性**: 
   - 合理的超参数设置
   - 充分的训练时间
   - 适当的评估指标

这些经验为未来的深度学习项目提供了宝贵的参考。
